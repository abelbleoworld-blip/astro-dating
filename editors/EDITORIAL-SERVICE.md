## Задача

Постоянная редакционная пара для всех текстовых артефактов экосистемы. Каждый редактор делает «свой проход» с цветовой разметкой; финал — пересечение их рекомендаций. Применимо к письмам, рукописям, лендингам, рецензиям, презентациям.

## Роли

### 🔵 Б. Акунин — литературно-стилевая правка

**Базовая специализация:** академический регистр, точность, лаконичность, культурная выверенность. Снимает «воду», литературные вставки, эмоциональные обороты.

**Доступные направленности (можно комбинировать):**

- **академический** (по умолчанию для писем экспертам, peer-review)
- **публицистический** (для лендингов, статей в журналах вроде «Кванта», «Природы»)
- **художественный** (для книги «Хронозвёзды», аудиокниги, литературных эссе)
- **юридический** (для оферт, договоров — точность формулировок, ссылки на статьи)

### 🟠 M. Componi — структурно-научная правка

**Базовая специализация:** даты, цитаты, факты, консистентность чисел между документами, проверка ссылок и идентификаторов (DOI, ORCID, ISBN), маркировка точек верификации.

**Доступные направленности (можно комбинировать):**

- **академический** (стандарт цитирования, унификация формул, проверка ссылок)
- **научно-популярный** (упрощение терминологии без потери смысла, добавление аналогий, контекстуализация)
- **адаптивно-универсальный** (один документ — несколько целевых аудиторий: учёные, журналисты, инвесторы, школьники; разные слои изложения)
- **техническо-документационный** (RFC-стиль, спецификации, README)
- **маркетинговый** (для лендингов, продуктовых описаний — продающий, но без буллшита)

## Формат вывода

При запросе «привлечь редакторов»:

1. **HTML-версия** в стиле track-changes (`EDITORIAL-REVIEW.html`): оригинал → правки Акунина (синий) → правки Компони (оранжевый) → финал
2. **Markdown финал** (`<NAME>-FINAL.md`) — чистая версия для отправки/публикации
3. **Сводная таблица правок** в начале HTML — что и кто менял
4. **Открытые вопросы** для верификации перед публикацией

Каждый артефакт сохраняет три состояния: original, edited, final.

## Расширение: рецензии (peer-review)

Тот же дуэт может **писать рецензии на статьи** — свои или чужие. Это особенно ценно для:

- внутренней валидации SOL-1 / CIV-1 / WorldRank статей перед подачей в журналы
- формирования датасета «параллельных рецензий в разных стилях»

### Формат рецензии

| Раздел       | Что включает |
|---|---|
| **Резюме**   | 1–2 предложения о работе |
| **Сильные стороны** | пункт-список, конкретные примеры с цитатами |
| **Замечания** | категории: critical / major / minor; со ссылками на разделы рукописи |
| **Методологические вопросы** | особенно по статистике, цитированию, воспроизводимости |
| **Рекомендация** | accept / minor revisions / major revisions / reject — с обоснованием |
| **Раскрытие конфликта интересов** | стандарт |
| **Стиль рецензии** | academic / popular / adaptive (выбирается заказчиком) |

Один редактор пишет рецензию в одном стиле; пара даёт две версии (Akunin-academic + Componi-popular, например). Это базовый материал для датасета.

## Расширение: датасет «параллельных рецензий»

**Концепция.** Один и тот же текст (статья, оферта, лендинг) проходит две независимые редакции / рецензии в разных стилях. На выходе — структурированный датасет:

```jsonl
{"source_id":"sol1-paper-v2", "editor":"akunin", "style":"academic", "review":"...", "tone":"formal", "recommendations":[...]}
{"source_id":"sol1-paper-v2", "editor":"componi", "style":"popular", "review":"...", "tone":"educational", "recommendations":[...]}
{"source_id":"sol1-paper-v2", "editor":"akunin", "style":"publicist", "review":"...", "tone":"engaging", "recommendations":[...]}
{"source_id":"worldrank-method", "editor":"componi", "style":"adaptive", "review":"...", "tone":"layered", "recommendations":[...]}
```

**Применения датасета:**

1. **Обучение editor-моделей** — стилевой transfer, scientific paraphrase, audience adaptation
2. **Бенчмарк** для редакторских/peer-review LLM
3. **Продажа через dataset-shop** — ниша «multi-style scientific editing» уникальна, конкурентов нет
4. **Внутреннее использование** — все будущие тексты экосистемы проходят через эту пару

**Минимальный объём для продажи (MVP):**
- 10 статей × 2 редактора × 3 стиля = 60 рецензий
- Каждая ~500–1500 слов
- Параллельные структурированные метаданные (source, editor, style, recommendations)
- Лицензия CC-BY-NC-4.0 (для коммерции — отдельная)
- Цена: 25 000 ₽ (выше WorldRank metrics-217, ниже multilingual corpus)

## Связь с экосистемой

| Артефакт | Где | Кто использует |

| Книга «Хронозвёзды» | `astro-dating/book/` | литературная редакция |

## Запуск редакционной сессии

Команда: «привлеки Акунина и Компони, стиль X, на документ Y»

Пример:
> «Акунин научно-популярный, Компони адаптивно-универсальный, на `astro-dating/paper/sol1.tex`»

→ выходит:
- `editors/sol1-akunin-popular.md`
- `editors/sol1-componi-adaptive.md`
- `editors/sol1-EDITORIAL-REVIEW.html` (визуальный track-changes)
- `editors/sol1-FINAL-popular.md` или `sol1-FINAL-adaptive.md` (выбор финала)
