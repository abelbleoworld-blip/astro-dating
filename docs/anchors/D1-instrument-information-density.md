# D1 — Информационная плотность древних астрономических инструментов

**Класс якоря:** новый — *информационно-инструментальный* (продолжение линии Альмагеста A4).
**Статус:** скелет, метод не реализован.
**Дата:** 2026-04-17

## Тезис

Древние астрономические инструменты можно охарактеризовать единой метрикой — **числом бит, которое инструмент способен закодировать или предсказать**. Эта метрика никем систематически не считалась. Если её посчитать для известного корпуса, получим **кривую информационной плотности по эпохам**, на которой можно видеть скачки технологии.

## Объекты для измерения

| Артефакт | Эпоха | Что кодирует | Грубая оценка бит |
|---|---|---|---|
| Стоунхендж (heel stone, sarsen circle) | ~3000 BCE | дискретные азимуты восхода/захода | log₂(50 камней) ≈ **6 бит** |
| Дрезденский кодекс Майя | ~XII в. CE | таблицы Венеры, затмений | ~3 бит/символ × ~78 страниц ≈ **9 кбит** содержательной части |
| MUL.APIN (вавилонские таблички) | ~1000 BCE | каталог 66 звёзд + сезонные восходы | log₂(66) × 3 параметра ≈ **18 бит** на звезду |
| Антикитерский механизм | ~150 BCE | планетные циклы, Saros, Метон | оценка по числу зубьев и редукций ≈ **10 бит** точности предсказания |
| Армиллярная сфера | ~250 BCE | 3D небесная сфера | ~8 бит угловой точности |
| Каталог Альмагеста (Гиппарх/Птолемей) | ~−130..+137 | 1022 звезды × (RA, Dec, mag) | ~10 бит/координата × 2 × 1022 ≈ **20 кбит** |
| Астролябия (Птолемеева) | ~150 CE | звёздная карта + календарь | ~6 бит звёздной точности |
| Мехаон (Антикитерский апгрейд?) | гипотетический | — | — |

Цифры выше — **первичная прикидка**, не результат. Метод нужно стандартизировать.

## Метод

1. **Определение «бита» для аналогового устройства.** Для дискретного (Дрезден) — log₂(размер алфавита) × длина. Для аналогового (астролябия, армилляры) — log₂(угловая точность / диапазон). Для механического предсказателя (Антикитерский) — log₂(временной горизонт точного предсказания / минимальная различимая фаза).
2. **Каталогизация.** Свести в таблицу: артефакт, эпоха ±, источник реконструкции, информационная ёмкость, функциональная роль (запись / расчёт / предсказание).
3. **Кривая по эпохам.** Построить log(бит) от века. Ожидаем нелинейные скачки на ключевых эпохах: позднеэллинистическая (Антикитерский, Альмагест), Сун (китайские инструменты), исламский ренессанс (астролябии XI-XIII вв.).
4. **Cross-check на узлах.** Mac и Beelink реализуют формулы независимо, сверяют числа.

## Что это даёт проекту

- **Связь с A4.** Альмагест уже верифицирован как датируемый артефакт (Морозов-Дамбис). Здесь Альмагест встаёт в **общую шкалу информационных машин древности** — это контекст для цифры «20 кбит». Видно, что каталог такого размера — не аномалия позднего средневековья, а согласованное продолжение линии Гиппарх→Птолемей.
- **Аргумент против Фоменко (косвенный).** Если кривая информационной плотности **гладкая** и согласована с независимо датируемыми артефактами, модель «всё античное создано в средневековье» требует объяснить отсутствие промежуточных ступеней. У Фоменко такого объяснения нет.
- **Самостоятельная ценность.** Никто не публиковал такую кривую в систематическом виде. Это новый research, не зависящий от спора о хронологии.

## Ограничения

- Грубость метрики. «Бит» — приближение; функциональная сложность инструмента не сводится к информационной ёмкости (Антикитерский — это **алгоритм**, не таблица).
- Реконструкции (Антикитерский, Стоунхендж) спорны в деталях — числа будут меняться по мере новых находок.
- Сравнение разных эпох/культур одной шкалой — предмет критики; обязателен раздел «методологические оговорки».
