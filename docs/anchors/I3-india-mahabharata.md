## 1. Тезис I3 — индийская хронология через планетарные конфигурации
Класс: планетарно-хронологический якорь. Дата: 2026-04-18. Размер: 64 строки (4×16).
Махабхарата (Bhīṣma Parva) описывает планетарную конфигурацию перед битвой на Курукшетре.
Традиционная датировка: ~3102 BCE (начало Кали-юги). Альтернативы: ~1478 BCE, ~950 BCE.
Метод: кластерный анализ — перебор эпох (-4000..0 CE), для каждой проверяем 7 планет.
Конфигурация Махабхараты: Сатурн в Рохини, Марс ретроградный, Юпитер около Вишакхи и др.
Результат (из Подробного описания): кластер ~1478 BCE — лучшее совпадение (5/7 планет).
3102 BCE (традиционная) — слабое совпадение (2/7). Фоменко не применим (нет «Новой хронологии» для Индии).
4 этапа индийской астрохронологии: Махабхарата (I3), Рама (I3b — negative result ожидается),
Эпохи/юги (I3c — структурная, не астрономическая), Jyotisha Vedanga (I4 — прецессионная датировка).
Cross-match с другими якорями: ~1478 BCE близко к вавилонской шкале (MUL.APIN ~1200 BCE).
Если подтвердится — независимая синхронизация Индии с Месопотамией через астрономию.
Связь с CIV-1: индийская сиддханта Арьябхаты (499 CE) = data point на кривой цивилизации.
Связь с SOL-1: индийские наблюдения затмений → дополнительные якоря для solar calendar.
Astro-dating расширяется: 9 → 10 цивилизаций (Индия — первая неавраамическая полностью).
Текущие результаты: graph `03_mahabharata_cluster.png` в Подробном описании эксперимента.
## 2. Метод (кластерный анализ Махабхараты)
Шаг 1. Извлечь планетарные описания из Bhīṣma Parva (Crit. Ed. Poona 1970, шлоки 2.16-3.37).
7 планет: Солнце, Луна, Марс, Меркурий, Юпитер, Венера, Сатурн + накшатры (лунные стоянки).
Шаг 2. Для каждой пробной эпохи T ∈ [-4000, 0] год: рассчитать положения 7 планет через JPL DE422.
Преобразовать в индийскую систему (накшатры = 27 секторов по 13°20' эклиптики).
Шаг 3. Для каждой T: подсчитать совпадение описание↔расчёт по 7 планетам (0-7 баллов).
Фильтр: Mars ретроградный (обязательно) + Saturn в Rohini (обязательно) = 2 hard constraints.
Шаг 4. Максимум score по эпохам → лучший кандидат. Bootstrap: удаление 1 планеты → устойчивость.
Реализация: `src/india_mahabharata.py` (≤256 строк), skyfield + DE422, шаг 1 год.
Mesh peer-review: Mac рассчитывает, Beelink проверяет независимо.
Протокол идентичен E1 Dendera (перебор эпох × планетная конфигурация).
Ожидаемое время: ~5 минут на полный перебор 4000 эпох × 7 планет.
Ключевое отличие от E1 Dendera: Махабхарата даёт РЕТРОГРАДНОСТЬ (динамический marker).
Ретроградность = сильнейший constraint (окно ~3 месяца из ~2 лет для Mars).
Вычислительная сложность: O(N_epochs × N_planets) = O(4000 × 7) = тривиально.
Данные: JPL DE422 (уже в repo: de422.bsp), Bhīṣma Parva (Poona Critical Edition 1970).
## 3. Литература и ограничения
1. Mahābhārata. Bhīṣma Parva, Crit. Ed. BORI Poona 1970 (шлоки 2.16-3.37 = планеты).
2. Achar B.N. *Date of the Mahābhārata War based on astronomical data*. Indian J. Hist. Sci. 2012.
3. Rao N.K. *Astronomical dating of the Mahābhārata War*. Current Science 2004.
4. Sengupta P.C. *Ancient Indian Chronology*. University of Calcutta 1947.
5. Thompson R. *Vedic Cosmography and Astronomy*. Bhaktivedanta Book Trust 1989.
Ограничение 1: перевод накшатр в эклиптические долготы — неоднозначен (несколько таблиц).
Ограничение 2: литературный текст ≠ точное наблюдение (Махабхарата = эпос, не дневник).
Ограничение 3: «negative result» (Рама 5561 BCE) = ожидаемый для столь древней даты.
Ограничение 4: индийская хронология вне Saros/Halley coverage → нет independent cross-match до -240.
Митигация: preregistration constraints (Mars retro + Saturn Rohini) ДО перебора.
Митигация: если нет совпадения > 5/7 → honestly report as «inconclusive».
Связь с D1: Jyotisha Vedanga = information-density точка (сиддханта ~500 CE, ~360 бит).
Связь с L8 multi-fractal: индийская хронология = Fibonacci-scaled юги (4:3:2:1 пропорция).
