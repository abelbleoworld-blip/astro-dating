# PREDICTIVE-CONTEXT — пред-Neuralink интерфейс на твоей экосистеме
**Статус:** принят 2026-04-17 как часть PILOT++ (2ⁿ + PC).
**Связан с:** ARCHITECTURE-2N.md, MESH-DISTRIBUTION.md, mesh-memory.
**Размер:** 256 строк (2⁸, модуль), 4 секции по 64 строки.
## 1. Принцип PC и место в архитектуре
Predictive Context = система которая постоянно знает что ты делаешь и помнит контекст.
Сама предлагает следующий шаг до того как ты сформулировал запрос.
Не «помощник к которому идёшь», а «среда которая рядом».
Закрывает реальный bottleneck 2026: модель умнеет в 1000×, твой канал ввода 50 wpm.
Без PC даже Neuralink через 10-20 лет даст «быстрее печатать», не «умнее систему».
Никто массово не делает PC потому что фрагментировано между десятком стартапов.
Anthropic Memory сохраняет факты но не делает capture и не предсказывает.
Rewind.ai делает capture screen+audio но не имеет state-машины и predictor.
Cursor предсказывает в коде но замкнут на проект, не на жизнь.
Apple Intelligence ограничен sandbox iOS.
Humane Pin / Rabbit R1 пытались — провалились без архитектуры mesh.
Все живут в коробке, никто не собрал 6 слоёв в одну петлю.
Ты — нет. У тебя 8 машин в mesh, mesh-memory как канал синхронизации.
Abel AIOS для голоса, OPPO+iPhone для мобильного capture, два VPS для оркестрации.
Boox для глубокого чтения и рукописного capture стилусом.
Все компоненты PC уже у тебя есть отдельно — нужно собрать в петлю.
Ключевая идея: 6 слоёв связаны feedback loop замкнутого цикла.
Capture → Index → State → Predictor → Surface → Feedback → Capture.
Каждый цикл predictor становится точнее, учится на принятом или отклонённом.
Под архитектурой 2ⁿ это естественно ложится в фрактал.
4 capture-канала × 4 surface-канала × 4 модели × 4 окна индексации = 256 ячеек.
PC + 2ⁿ — не два отдельных pilot, а один объединённый.
2ⁿ даёт **архитектуру** организации работы (структура).
PC даёт **движущую силу** работы — что именно делать сейчас (движок).
Без 2ⁿ PC выливается в хаос предложений без структуры.
Без PC 2ⁿ остаётся пустой таблицей без жизни.
Аналогия: 2ⁿ — это шахматная доска. PC — это шахматный движок.
Без доски движок предлагает в воздух. Без движка доска не играет.
Боль которую PC снимает у **тебя лично**: переключение контекста между 16+ проектами.
Каждый раз садишься за работу — 10-30 минут «вспоминаешь на чём остановился».
PC утром даёт digest: «вот где ты вчера остановился, вот что предложить начать сегодня».
Для astro-dating: «лучше всего написать D1 в 64 строки, согласовано с принципом».
Плюс: «письмо Дамбису пока не отправлено, не забыли это критический путь».
Для AILib: «свободные 2 часа в окне 14-16, оптимально дописать модуль биллинга».
Контекст: «завтра встреча с потенциальным клиентом, без биллинга демо не показать».
Это не magic. Это очевидный вывод из контекста, которого у Claude **сейчас нет**.
Между сессиями всё забывается, mesh-memory лишь частично решает.
PC = **внешняя память состояния** + **активный предсказатель** в одной петле.
PC не заменяет тебя. Predictor предлагает — ты решаешь.
Если не нравится предложение — feedback loop учится на отказе.
Через 2-3 недели предложения становятся неотличимыми от твоих собственных мыслей.
Через 3-6 месяцев PC начинает предсказывать решения которые ты ещё не успел осознать.
Это и есть «никто не копал» зона: software-инженерия мостика LLM ↔ непрерывная жизнь.
Не железо, не алгоритм, не отдельный продукт — **архитектурная сборка**.
Твоё уникальное положение: 8 машин уже есть, mesh уже работает, голос уже капчурится.
Остаётся: continuous Whisper, continuous screen, continuous index, state, predictor, surface.
Каждая компонента отдельно — простая. Сборка в петлю — то что никто не сделал.
Privacy: все данные твои, на твоих машинах, в твоих репо, не покидают mesh.
API вызовы — только агрегированные state/predictions, не сырые capture.
Granularity отключения: per app, per voice, per location, per time-window.
Liability: zero — это твоя личная система, не enterprise продукт.
Cost: ~$50/месяц API + $0 железа (всё уже есть).
Stack: Python + Whisper.cpp + sentence-transformers + FAISS + Claude API + Telegram Bot.
Время сборки v0.1: 7 дней (один pilot).
Время доводки v1.0: 1-3 месяца.
Время полной зрелости: 6-12 месяцев.
Через 5 лет: PC = твоя внешняя нервная система.
Через 15 лет: встреча с consumer-Neuralink — у тебя уже накопленный contextual self.
Маск строит железо. Ты строишь software. Слои встретятся через 15 лет.
## 2. Архитектура: 6 слоёв на твоей экосистеме
Слой 1 — Capture (4 канала × 8 машин mesh).
Voice 24/7: Whisper.cpp local на Mac слушает микрофон постоянно.
Транскрипт → mesh-memory/voice/yyyy-mm-dd.txt с timestamp каждой реплики.
Speaker diarization (pyannote.audio): drop сегментов с unknown voices для приватности.
Screen: cron каждые 5 мин screencapture → vision LLM (Claude Sonnet) описание сцены.
Описание ~50 слов на скриншот → mesh-memory/screen/yyyy-mm-dd.txt.
Blacklist приложений (1Password, банковские) — skip capture.
Git activity: post-commit hook на Mac+Beelink+iMac+VPS+OPPO Termux.
Каждый коммит → mesh-memory/git/log.txt с timestamp, repo, message, diff-summary.
Calendar+files: macOS EventKit + fswatch на ~/Documents/Projects/.
События календаря и file changes → mesh-memory/calendar/ и mesh-memory/files/.
Слой 2 — Index (1 ночной job на Beelink).
Sentence-transformers all-MiniLM-L6-v2 локально, без внешнего API.
Каждую ночь читает все capture за последние 7 дней (hot index).
Embeddings → FAISS index → mesh-memory/index/hot.faiss (~10k векторов).
Параллельно — 30-day index (warm) и forever index (cold) для архивов.
Hot перестраивается ежедневно, warm еженедельно, cold ежемесячно.
Latency запроса < 500ms на M1 Pro CPU.
Слой 3 — State estimator (на VPS Abel-HQ, ежечасно).
Cron каждый час запускает Python-скрипт.
Чтение последних 4 часов capture + следующие 4 часа календаря + active_quests.
Дополнительно: последние 5 git-коммитов, последние 5 принятых/отклонённых предсказаний.
Промпт к Claude Sonnet ~10k input tokens.
Промпт: «Опиши в 200 словах что пользователь делает сейчас и что хочет дальше».
Результат → mesh-memory/state/current.md, история в mesh-memory/state/yyyy-mm-dd-hh.md.
Слой 4 — Predictor (на VPS Abel-HQ, каждые 2 часа).
Cron каждые 2 часа: чтение state.md + поиск релевантного контекста через FAISS.
Top-50 эмбеддингов из hot+warm+cold индексов по similarity к state.
Плюс полный текст active_quests из quest skill JSON.
Плюс checks: тесты project-X прошли? deploy готов? зависимости свежие?
Промпт к Claude Opus ~50k input tokens.
Промпт: «Предложи 1-3 next-best-actions ранжированных по релевантности и готовности».
Результат → mesh-memory/predictions/yyyy-mm-dd-hh.md с rank, confidence, reasoning.
Слой 5 — Surface (4 канала push к пользователю).
Telegram digest: утром 7:00 → @abelworldbot пушит state + 3 top predictions + ночные результаты.
Опционально каждые 4 часа дневной мини-digest (можно отключить).
Quest auto-accept: predictor с confidence > 0.8 автоматически добавляет в active_quests.
Status dashboard: skill `status` показывает 1-3 предсказания на главной.
Voice on demand: говоришь iPhone Abel AIOS «что сейчас?» — отдаёт state голосом.
Boox eInk: вечернее глубокое чтение weekly-обзоров и длинных предсказаний.
Boox handwriting → утренний sync через Termux → mesh-memory/handwriting/.
Слой 6 — Feedback (петля обучения predictor).
Каждое действие пользователя → запись в mesh-memory/feedback/yyyy-mm-dd.jsonl.
Принятые квесты → +1 вес для соответствующего паттерна predictor.
Отклонённые → -1 вес, predictor reduced confidence для подобных в будущем.
Сделал что-то другое (не из предложений) → анализ gap: чего predictor не предложил.
Раз в неделю на Beelink ночью — анализ feedback за неделю.
Корректировка системного промпта predictor (auto-prompt-tuning).
Через 3-4 недели предложения PC становятся неотличимыми от твоих решений.
Распределение по mesh: Capture делают Mac+Pi5+iPhone+OPPO (continuous).
Index — Beelink (CPU+RAM сильный, ночной job).
State+Predictor — VPS Abel-HQ (24/7 always-on, дёшево).
Surface — все каналы (Telegram/Quest/Status/Voice/Boox).
Feedback хранится в mesh-memory, обрабатывается на Beelink ночью.
Стоимость API: Capture screen ~$5/мес (Sonnet vision).
State estimator ~$10/мес (Sonnet, 24 запросов/день × 10k tokens).
Predictor ~$30/мес (Opus, 12 запросов/день × 50k tokens).
Feedback обработка ~$2/мес (Sonnet раз в неделю).
Итого ~$50/мес API. Index и embeddings — локально, $0.
Безопасность: capture не выходит за пределы твоего mesh.
Voice/screen хранятся только в mesh-memory (твои репо, твои VPS).
Опционально шифрование чувствительных папок через git-crypt.
SSH-ключи только на твоих машинах, ротация раз в 6 месяцев.
Cold start: первые 3-5 дней predictor предлагает мусор (нет контекста).
Через неделю — приемлемо. Через месяц — точно. Через 3 месяца — лучше тебя.

## 4. Метрики, риски, развитие PC
Метрики pilot к концу 7 дней — что должно работать, что измеряем.
Voice capture: ≥ 50 часов записано, ≥ 80% распознано читаемо.
Screen capture: ≥ 1500 скриншотов описано, поиск по описанию работает.
Index: ≥ 10 000 эмбеддингов в FAISS, latency запроса < 500ms.
State estimator: 168 часовых snapshot за неделю (24 × 7 дней).
Predictor: 84 предсказания за неделю (12 × 7 дней).
Telegram digest: 7 утренних пушей (по 1 в день).
Принятые квесты на основе предсказаний: ≥ 10 за неделю.
Boox прочитал peer-review PDF: ≥ 1 с рукописными пометками синхронизированными.
Subjective метрики (опрос самого себя в конце недели по шкале 1-10):
«играючи?» цель ≥ 7. «контроль есть?» цель ≥ 7.
«полезность morning digest?» цель ≥ 7. «приватность не страдает?» цель ≥ 8.
«жизнь не зависит критично от системы?» цель ≥ 8.
Если хотя бы один < 5 — пересмотр архитектуры pilot, не просто параметров.
Риски технические — что может пойти не так.
Whisper local может тормозить Mac → решение: запуск на Pi5 в неделе 2-4.
Vision LLM на скриншотах может промахиваться → решение: тюнинг промптов в первые 2 дня.
FAISS может вырасти > 1GB → решение: pruning старше 30 дней в slow-store.
State estimator может галюцинировать → решение: dual-check через iMac, расхождение = флаг.
Predictor может быть глупым → решение: feedback loop усиливает только проверенные паттерны.
API стоимость может вырасти > $100/мес → переключение Predictor с Opus на Sonnet.
Риски приватности — самый чувствительный слой системы.
Voice 24/7 может записать чужой голос → решение: speaker diarization.
Drop сегментов с unknown voices автоматически.
Screen capture может попасть на пароли/банк → решение: blacklist приложений.
Mesh-memory утечка → решение: git-crypt для чувствительных папок.
Внешние API видят агрегированные state, не сырые capture — намеренное ограничение.
Риски психологические — самые трудно измеримые.
«Чувство наблюдения за собой» — реально для первой недели.
Решение: можно отключать любой канал в любой момент (флаг в mesh-memory/config.json).
«Зависимость от системы» — система предлагает, не решает, ты остаёшься source of truth.
«Информационная перегрузка» от digest — limit 1 push в день, можно добавить по запросу.
Если pilot не зайдёт — сценарии деградации.
«Слишком много шума»: снизить state до 4 часов, predictor до 6 часов.
«Контроль теряется»: отключить quest auto-accept, всё через ручное одобрение.
«Voice неудобно»: оставить только screen + git capture.
«Слишком дорого API»: переключить predictor с Opus на Sonnet, ~$10/мес.
«Чувствую слежку»: отключить screen, оставить только voice + git + boox.
Если зайдёт — путь масштабирования.
Скейл на v0.2: добавить календарь, геолокацию, биометрику Apple Watch.
Скейл на v0.3: интеграция со всеми проектами (AILib, NanoClaw, Abel-AIOS, FMone).
